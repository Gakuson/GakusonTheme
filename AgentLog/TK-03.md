# TK-03 ヘッダー検索モーダル

## Goal
- 既存ヘッダーの検索導線を、`searchform.php` ベースの WordPress 標準検索フローへ接続し、PC/SP のモーダル開閉と ARIA 状態を整理する。

## 変更内容
- `header.php` の PC 検索 UI を、常時表示 input から検索トリガーボタンへ変更し、共通の検索パネル `#header-search-panel` を追加した。
- `header.php` の SP 検索ボタンも同じ検索パネルを開く構成に揃え、ハンバーガーボタンには `aria-controls` / `aria-expanded` を追加した。
- `searchform.php` を新規追加し、`form role="search"`、`s`、`category_name`、`tag` を持つ共通フォームを実装した。
- `searchform.php` ではカテゴリとタグを単一選択 `<select>` として出力し、現在の GET 値があれば再表示時に選択状態を復元するようにした。
- `js/script.js` のヘッダー UI 制御を整理し、検索パネルとメニューを排他的に開閉するようにした。
- `js/script.js` で検索パネルの `aria-hidden` / トリガー側 `aria-expanded`、閉じるボタン、Escape キー、フォーカス復帰を実装した。
- `js/script.js` でハンバーガーメニューの `aria-expanded` の向きを正し、存在しない `.navSp_dropdown` 参照と `"ture"` typo を解消した。
- `smacss/module/module.scss` と `smacss/state/state.scss` に、検索トリガー、検索パネル、フォーム項目、開閉状態の最小スタイルを追加した。
- `smacss/main/*.css`、`smacss/module/module.css`、`smacss/state/state.css` は、上記 SCSS に対応する出力へ同期した。

## 判断理由
- design/tickets の「検索入口はヘッダーモーダル」「`get_search_form()` / `searchform.php` と整合」に合わせ、PC/SP で別々の input を持つ構成ではなく、1 つの検索パネルを共有する形に寄せた。
- 既存ヘッダー構造は大きく崩さず、`header_search` / `header_spMenu` / `slideInput` など既存 class と配置をできるだけ残したまま、検索の中身だけを WordPress 標準フォームへ差し替えた。
- `TK-03` は検索 UI 接続の責務に留め、検索結果テンプレート、`pre_get_posts`、`post` 限定ロジックは `TK-04` に残した。
- ARIA とフォーカス制御は検索モーダルだけでなく、同じ `js/script.js` 内にあるハンバーガーメニューの状態管理も一緒に直した方が、検索とメニューの競合を減らせると判断した。
- Sass CLI は sandbox からグローバル package 実体への参照で `EPERM` になり通常ビルドできなかったため、SCSS を正本として更新した上で、生成 CSS は該当断片だけ局所同期した。
