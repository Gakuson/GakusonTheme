# TK-11 gdocs_to_wordpress 入稿アプリ対応

## ticket id and title
- TK-11 gdocs_to_wordpress 入稿アプリ対応

## goal
- `docs/theme_development_guide.md` の入稿アプリが出力する HTML を、このテーマの投稿ページと固定ページで崩さず表示できるようにする。
- 目次、吹き出し、見出し、引用、テーブル、画像に加えて、太字、下線、色付きテキストなどのインライン装飾も本文中で自然に見える状態にする。

## 変更内容
- `smacss/module/_content-entry.scss`
- gdocs_to_wordpress 向けの本文スタイルを共通モジュールとして新設した。
- `span[style]` と装飾用 span を `.content` 内で inline に戻し、グローバルの `span { display: block; }` の影響を本文だけで打ち消した。
- `h2` / `h3` / `h4` の `wp-block-heading` をレベルごとに分けて調整し、目次アンカー、引用、リスト、テーブル、画像、吹き出し、リンク装飾を共通化した。
- `smacss/page/page-post.scss`
- 投稿本文の見た目をラッパー中心に整理し、詳細な本文装飾は共通モジュールへ移した。
- `smacss/page/page-fixed.scss`
- 固定ページ本文にも同じ共通モジュールを適用できるよう、ラッパーの体裁を整えた。
- `smacss/main/main-post.scss`
- 投稿バンドルへ共通本文モジュールを追加した。
- `smacss/main/main-fixed.scss`
- 固定ページバンドルへ共通本文モジュールを追加した。
- `smacss/main/main-post.css`
- ビルドを実行して投稿向けの配信 CSS を更新した。
- `smacss/main/main-fixed.css`
- ビルドを実行して固定ページ向けの配信 CSS を更新した。

## 判断理由
- ベースの `span { display: block; }` を全体で外すと既存レイアウトへの影響が大きいため、入稿アプリの HTML が置かれる `.content` の内側だけで補正する方が安全だった。
- 投稿と固定ページの両方で同じ本文 HTML を扱うため、ページ別に重複実装するより共通モジュール化した方が保守しやすいと判断した。
- `wp-block-heading` を一括指定すると `h3` / `h4` まで `h2` 相当の装飾を拾ってしまうため、レベル別の明示指定へ直した。
