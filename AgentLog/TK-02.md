# TK-02 WordPress標準出力ベース整理

## Goal
- 既存テンプレートを大きく崩さず、WordPress 標準出力を後続チケットの styling hook として使える状態へ寄せる。

## 変更内容
- `front-page.php`、`single.php`、`page.php`、`category.php`、`tag.php`、`page-newindex.php` に残っていた template-local な `wp_tag_cloud` filter を撤去した。
- tag cloud は `gakuson_get_tag_cloud_args()` と `gakuson_format_tag_cloud_markup()` を直接使う形へ置き換え、`wp-tag-cloud` / `tag-cloud-link` などの WordPress 標準 class を残したまま、既存の `tag_list` / `Hashtag_text` / `tag_itemLink` も併用するようにした。
- `inc/gakuson-content-helpers.php` に `gakuson_get_article_taxonomy_markup()` を追加し、front-page の記事カードで重複していたカテゴリ / タグ表示だけを小さく helper 化した。
- `post_class()` を front-page の記事カード、`single.php` / `page.php` の本文 article・関連記事カード、`category.php` / `tag.php` の一覧カード、`sidebar.php` のランキングカードに適用した。
- `tag.php` の見出しを `single_tag_title()` ベースへ修正し、壊れていた closing tag を整理した。
- `sidebar.php` の 3 件目だけ `feature_sidebar` になっていた class typo は、`post_class()` 適用に合わせて `feature__sidebar` に揃えた。

## 判断理由
- TK-01 で tag cloud 用の薄い helper が入っていたため、TK-02 では新しい専用 helper を増やさず、既存 helper を各 template から使う方針にした。
- front-page の taxonomy 表示は同一 markup の重複が多く、ここだけは「カード全体の partial 化」ではなく「繰り返し部分だけ helper 化」に留めるのが ticket の意図に合っていた。
- `post_class()` を入れても既存の `feature` / `article_item` / `l-article` class は残るため、現行 CSS を壊さずに WordPress 標準 class を後続チケット用の hook として追加できる。
- 今回の ticket は構造整理が中心で、見た目の横展開は TK-06 / TK-07 の責務なので、SCSS は変更せず markup と helper の整理に範囲を絞った。
