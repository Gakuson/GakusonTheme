# TK-01 基盤整備

## Goal
- 後続チケットの前提になる helper の置き場と責務を先に固定し、Sass の出力導線を現行テーマ構成へ合わせる。

## 変更内容
- `functions.php` から `inc/gakuson-content-helpers.php` を読み込むようにした。
- `inc/gakuson-content-helpers.php` を新規追加し、以下の共通 helper をまとめた。
- `featured` タグ記事取得
- カルーセル item / response 生成
- 抜粋、カテゴリ名、タグ名、サムネイル URL の共通整形
- transient cache key / TTL / read / write / delete
- `save_post_post`、`set_object_terms`、`before_delete_post` を使ったキャッシュ無効化
- `wp_tag_cloud()` 向けの共通引数 helper と markup 整形 helper
- `package.json` の Sass script を `smacss/main` 出力に合わせて更新した。
- `build:css`
- `watch:css`

## 判断理由
- template に helper を増やす方針は禁止なので、今後の横展開や endpoint 実装で再利用できる関数群を `inc/` に切り出した。
- `featured` 抽出とカルーセル payload shape を 1 箇所にまとめることで、`front-page.php` と `TK-08` の endpoint 実装が別々に仕様を持たない状態を先に作った。
- cache TTL は未確定事項なので、固定値を散らさず `gakuson_get_carousel_cache_ttl()` に閉じ込め、後で filter か実装変更だけで差し替えられる形にした。
- `wp_tag_cloud()` は template ごとに直書き filter が散っているため、`TK-02` で移設できるように汎用の整形 helper だけ先に置いた。
- Sass は `smacss/main/*.css` が WordPress で読まれているため、旧 `css:css` watch ではなく main directory compile に合わせた。

## Touched Files

- `inc/gakuson-content-helpers.php`

- カルーセル response の `tags[]` には現時点で `featured` タグも含まれる。表示時に除外したいなら `TK-05` か `TK-08` で明示判断が必要。
