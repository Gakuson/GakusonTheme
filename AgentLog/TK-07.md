## 変更内容
- `category.php` / `tag.php`
- archive 見出し直下に件数表示 `archivePage_count` を追加し、カテゴリ・タグ一覧の導線トーンを検索結果ページに揃えた。
- `smacss/module/module.scss`
- 共通の section heading、article card、tag chip に surface / shadow / accent line を追加し、トップと下層の見出し・カードの質感を同系統に寄せた。
- `smacss/module/sidebar.scss`
- sidebar section 自体を panel 化し、人気記事カードと外部リンクカードの hover / shadow / border をトップ基準のトーンへ寄せた。
- `smacss/page/page-post.scss`
- 単記事ヘッダーを白い panel + accent line に整理し、カテゴリ・タグ chip、summary box、本文 panel を調整した。
- 本文内のテキストリンクだけ `display:inline` に戻し、記事可読性が block link 化で崩れないようにした。
- `smacss/page/page-fixed.scss`
- 固定ページの本文と block title を同じ surface 系ルールに寄せ、関連記事導線とのトーン差を小さくした。
- `smacss/page/page-category.scss`
- archive / search の intro、条件 summary、再検索フォーム、empty state、pagination を同じ panel ルールで揃えた。
- 検索フォームは desktop で 3 カラム、tablet で 2 カラム、mobile で 1 カラムへ落ちるようにして、検索導線の密度を調整した。
- `smacss/main/main-top.css` / `smacss/main/main-post.css` / `smacss/main/main-fixed.css` / `smacss/main/main-category-tag.css` / `smacss/main/main-404.css`
- `npm run build:css` で再生成した。

## 判断理由
- TK-07 は「テンプレート統合」ではなく「見た目ルールの統一」が目的なので、HTML の大きな組み替えではなく、module/page 層の SCSS 調整を中心にした。
- 単記事は本文可読性優先のため、本文構造は維持したまま header / summary / taxonomy / surrounding rail を panel 化してトップの質感だけ寄せた。
- archive / search は同じ情報整理のページなので、件数表示・summary panel・pagination を共通トーンへ寄せると差分が減ると判断した。
- global に `a { display: block; }` が効いている既存テーマなので、記事本文と固定ページ本文だけは inline link に戻して読み心地の後退を避けた。
- tag directory は helper が public tag のみを出す設計なので、実データ側で public tag が無いページでは非表示のままにした。
