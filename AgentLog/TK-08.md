# TK-08 REST endpoint / direct fetch

## ticket id and title
- TK-08 REST endpoint / direct fetch

## goal
- `nantopi-pick` タグ付き記事の最新1件を、cross-origin で読める軽量 JSON として公開し、別サイト側から `fetch()` で利用できる状態にする。

## 変更内容
- `inc/gakuson-content-helpers.php`
- `nantopi-pick` 用 tag slug helper、投稿取得 helper、payload builder、transient cache helper を追加した。
- `GET /wp-json/gakuson/v1/picks` を `register_rest_route()` で追加し、public GET の JSON 応答を返すようにした。
- picks payload は `items[]` 配下に `title`、`tags[]`、`image`、`url` だけを返す構成にした。
- picks payload では制御タグ `nantopi-pick` だけを除外し、`isKk` を含む通常タグはそのまま返すようにした。
- `GAKUSON_PICKS_ALLOWED_ORIGINS` と `GAKUSON_PICKS_CACHE_TTL` を読める helper を追加し、picks route 専用の CORS allowlist と TTL 上書きに対応した。
- 既存の保存・ターム変更・削除フックで、carousel cache に加えて picks cache も無効化するようにした。
- `docs/3-17-yuki-design.md`
- TK-08 の route、payload、設定定数、`nantopi-pick` / `isKk` の扱いを現行仕様へ更新した。
- `docs/3-17-yuki-tickets.md`
- TK-08 の目的、対応内容、Done Criteria、確認事項を現行仕様へ更新した。
- `docs/3-17-gakuson-integration-design.md`
- 補足メモ側も `carousel` 前提から `picks` / `nantopi-pick` 前提へ更新した。

## 判断理由
- 既存 `featured` カルーセルはトップ用 UI としてすでに責務が固まっているため、TK-08 は parallel な `nantopi-pick` 系 helper を足して責務を分離した。
- `nantopi-pick` は公開 UI では見せたくない制御タグなので、既存の internal-only tag ルールへ加えつつ、API では picks payload 専用に除外条件を上書きして `isKk` などの通常タグは返す形にした。
- CORS は route 専用 allowlist に絞り、WordPress 全体の cross-origin 設定を広げない方針にした。
- 本番 origin は repo 内で確定できないため、theme 側は `wp-config.php` 定数を読む実装に留め、ローカル確認用だけ fallback origin を持つ形にした。

## Review notes / risks
- この repo には「別サイト側の静的 HTML / JS」は含まれていないため、`fetch()` して描画する consumer 実装自体は今回入れていない。
- `GAKUSON_PICKS_ALLOWED_ORIGINS` を本番で未設定のまま使うと、default の localhost origin 以外には CORS が開かない。
- `image` はアイキャッチ未設定時に空文字を返すだけで、placeholder URL の決め打ちは入れていない。

- ワークツリーには TK-08 着手前から多数の未コミット差分があり、今回は TK-08 関連ファイルだけに重ねている。
