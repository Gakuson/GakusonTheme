# TK-04 検索結果ページ

## ticket id and title
- TK-04 検索結果ページ

## goal
- WordPress の検索結果として、キーワード・カテゴリ・タグ条件を保持した結果ページを追加し、ヘッダー検索モーダルからの `s` / `category_name` / `tag` をそのまま結果表示へつなぐ。

## 変更内容
- `functions.php`
- `is_search()` でも `smacss/main/main-category-tag.css` を読み込むようにし、検索結果ページを category/tag と同じ一覧トーンで表示できるようにした。
- `gakuson_customize_search_query()` を追加し、main search query を `post` のみに制限したうえで、`category_name` と `tag` の単一選択フィルタを `pre_get_posts` で適用するようにした。
- `search.php`
- 新規追加し、現在条件表示、再検索フォーム、結果一覧、0件メッセージ、ページングを実装した。
- 結果一覧は既存の `feature-wrapper` / `feature` カードを使い、 sidebar と hashtag セクションも category 系テンプレートに寄せて配置した。
- `smacss/page/page-category.scss`
- 検索結果ページ専用の最小スタイルを追加し、条件チップ、再検索フォーム、0件表示、ページングを白ベースのカードとして収めた。
- `smacss/main/main-category-tag.css`
- Sass ビルドを再実行し、検索結果ページ向けの出力CSSを更新した。

## 判断理由
- TK-04 は「検索結果ページ追加」が責務なので、ヘッダー検索フォーム本体には触れず、TK-03 の `searchform.php` をそのまま再利用する方針にした。
- category/tag と近い一覧構造だったため、新しいコンポーネント抽出は行わず、既存マークアップと SMACSS レイヤーの範囲で検索結果ページだけを増やした。
- クエリ制御は template-local な `WP_Query` ではなく `pre_get_posts` に寄せ、URL リロードやページング時も同じ条件が維持される前提を優先した。
